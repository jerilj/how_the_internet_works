HTTP and HTTPS protocols





HTTP, Hypertext Transfer Protocol, is the protocol used for transmitting data between browsers and web servers. It is the most widely used protocol. It is primarily used for viewing web pages. In standard HTTP all information is sent as plain text. However this communication can be hijacked by attackers. Inorder to protect sensitive data sent through the internet,Secure Hypertext Protocol must be used. HTTPS will encrypt the data that is being transferred . HTTPS protocol makes sure the destination address is actually verified. This is done using an encryption algorithm before the transfer begins.

HTTPS protects the data by using SSL protocol layer. SSL uses public key encryption to protect. The browser will receive an SSL certificate from the web server.The information sent through the internet can be encrypted to avoid a third party from reading the information. A unique key is required to decrypt the information by the receiver. This is used to make sure the website is trustworthy. If the certificate is trustworthy an SSL session is established between the browser and the web server. TLS is another protocol like SSL. It is the successor to SSL. SSL and TLS protocols use public key encryption to secure the data.
